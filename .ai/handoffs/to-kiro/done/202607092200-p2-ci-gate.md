## Why
Today the merge gate was human-eyeballed (someone ran 6 commands + read the
output). ADR-0007 P2: make it a machine-checkable required check. On a clean CI
runner (git+node on PATH), the whole suite runs GREEN — no "known env-fail"
fudge.

## Task
Create `.github/workflows/gates.yml` — a GitHub Actions workflow that runs on
`pull_request` targeting `master` AND `push` to `master`, on `ubuntu-latest`,
with Node + git available. Steps (each a named step, fail the job on any red):
1. `bash .ai/tools/check-ssot-drift.sh` — expect Drift: 0
2. `bash .claude/hooks/test_hooks.sh`
3. `bash .kimi/hooks/test_hooks.sh`
4. `bash .kiro/hooks/test_hooks.sh`
5. `node .opencode/plugin/test-guard.mjs`
6. `bash scripts/git-hooks/test-pre-commit.sh` (the ADR-0005 backstop test)
7. `cd tools/multi-cli-install && npm ci && npx tsc --noEmit && npx vitest run`
   — on a real runner the 3 "git/npm not on PATH" failures should PASS; if any
   test genuinely fails, the gate must go red.
Add a short note to `.github/workflows/` (or the existing `framework-check.yml`
header comment) that branch protection must be enabled to make this REQUIRED
(that's a GitHub repo setting, not a file). Verify your YAML parses (a yaml
linter or `python -c 'import yaml,sys;yaml.safe_load(open(...))'`).
